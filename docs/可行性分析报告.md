# 学生宿舍信息管理系统可行性分析报告

## 1. 项目背景

学校宿舍管理需要维护学生住宿位置、宿舍电话、床位分配和调换申请。传统人工表格容易出现信息更新不及时、权限边界不清、统计困难等问题，因此需要一个集中化、可视化、可持久化的信息管理系统。

## 2. 技术可行性

系统采用 Java Web 方式实现，后端使用 JDK 自带 HTTP Server 提供 REST 接口，数据存储使用 MySQL 8.0，前端使用 HTML、CSS、JavaScript 构建企业后台界面。项目不依赖 Maven，启动脚本可自动下载 MySQL JDBC 驱动，便于课程答辩环境快速运行。

## 3. 数据可行性

MySQL 数据库包含楼栋、宿舍、床位、学生住宿、用户、调换申请、报修反馈和操作日志 8 张业务表。系统首次启动会自动创建数据库、数据表和演示数据，能够满足基础设施容量、住宿信息、账号权限、业务审批和操作审计的长期保存需求。

## 4. 操作可行性

系统提供管理员和普通用户两类入口。管理员可通过后台完成住宿台账、楼栋宿舍、账号、申请审批、报修处理、统计分析和操作日志管理；学生使用学号登录，只能查看本人住宿与同宿舍成员，并可提交调换申请和报修反馈。界面采用侧边栏、表格、指标卡、时间线和表单布局，符合常见企业后台操作习惯。

## 5. 经济可行性

系统基于 JDK、MySQL 和浏览器运行，无需购买商业软件或服务器。后续如需扩展，可进一步迁移到 Spring Boot、Vue 或 React 技术栈。

## 6. 结论

该系统技术路线清晰、运行成本低、功能覆盖课程设计要求，并具备较好的演示效果和扩展空间，项目可行。
