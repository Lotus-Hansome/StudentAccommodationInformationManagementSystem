# 学生宿舍信息管理系统需求分析报告

## 1. 系统目标

实现学生宿舍信息的集中管理，支持信息添加、修改、删除、查询、显示、排序、宿舍调换申请审批，以及基于统计数据的智能宿舍运营分析。系统采用 Web 企业后台界面和 MySQL 数据库存储，满足课程设计演示和实际管理场景。

## 2. 用户角色

| 角色 | 主要权限 |
| --- | --- |
| 系统管理员 | 添加、修改、删除、查询、显示、排序宿舍信息；查看和审批调换申请；查看统计数据和智能分析建议 |
| 普通用户 | 查询、显示、排序宿舍信息；提交和查看宿舍调换申请 |

## 3. 功能需求

1. 学生宿舍信息添加：录入学号、姓名、所在系、班级、宿舍号、宿舍电话号码、床位。
2. 学生宿舍信息修改：管理员可维护学生姓名、院系、班级、宿舍、电话和床位。
3. 学生宿舍信息删除：根据宿舍号和学号删除错误或已搬离学生记录。
4. 查询与显示：支持按宿舍号、学号查询，并能显示全部宿舍信息。
5. 排序：支持按所在系、班级、学号排序展示。
6. 宿舍调换申请：普通用户提交申请和理由；管理员审核同意或拒绝；同意后自动更新学生宿舍信息。
7. 数据看板：展示入住学生数、宿舍房间数、空余床位、待审申请、入住率和各系入住结构。
8. 智能宿舍分析：管理员查询楼栋或宿舍统计数据后，系统生成宿舍运营评估与建议；配置大模型接口时调用接口，未配置时使用本地规则分析。

## 4. 数据需求

学生宿舍信息包括：学号、姓名、所在系、班级、宿舍号、宿舍电话号码、床位、创建时间、更新时间。

调换申请信息包括：申请编号、学号、原宿舍、原床位、目标宿舍、目标床位、目标宿舍电话、申请理由、申请状态、申请时间、审核时间、审核意见。

## 5. 非功能需求

- 数据使用 MySQL 持久化保存，程序关闭后不丢失。
- 前端界面应美观、清晰，符合企业后台常见交互习惯。
- 系统需要进行服务端权限校验，普通用户不能执行管理员操作。
- 对重复学号、床位占用等错误进行校验。
- 智能分析模块需要支持真实接口和离线演示两种模式。
