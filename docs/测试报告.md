# 学生宿舍信息管理系统测试报告

## 1. 测试环境

- 操作系统：Windows
- JDK：17 及以上
- 数据库：MySQL 8.0
- 浏览器：Chrome/Edge 均可
- 默认访问地址：`http://localhost:8080`

## 2. 自动化测试

运行：

```powershell
.\run.ps1 -SmokeTest
```

预期输出包含：

```text
SMOKE_TEST_PASSED
```

本次已验证冒烟测试通过。

冒烟测试还会启动本地模拟的 OpenAI 兼容接口，校验统计数据请求体、模型名称和建议内容解析，不消耗真实模型额度。

## 3. Web 接口验证

已验证：

- 管理员 `admin/admin123` 登录成功。
- `/api/overview` 可读取 MySQL 概览数据。
- `/api/students?mode=all` 可读取 MySQL 学生住宿分页列表。
- 学生访问管理员接口返回 `403`，未登录访问受保护接口返回 `401`。
- Chrome 无头浏览器验证管理员和学生菜单隔离，页面无横向溢出。

验证结果：

```text
LoginSuccess=True
Role=ADMIN
TotalStudents=64
RoomCount=76
TotalCapacity=306
Buildings=12
ChangeRequests=10
```

## 4. 功能测试用例

| 编号 | 测试内容 | 操作 | 预期结果 |
| --- | --- | --- | --- |
| TC01 | 管理员登录 | 输入 `admin/admin123` | 进入企业后台页面 |
| TC02 | 普通用户登录 | 输入学号和初始密码 `student123` | 进入学生首页 |
| TC03 | 添加住宿信息 | 管理员填写完整学生住宿资料 | MySQL 保存成功，表格刷新 |
| TC04 | 修改住宿信息 | 管理员点击编辑并保存 | 对应学生记录更新 |
| TC05 | 删除住宿信息 | 管理员点击删除 | 记录从 MySQL 删除 |
| TC06 | 按宿舍查询 | 输入宿舍号查询 | 显示该宿舍学生 |
| TC07 | 按学号查询 | 输入学号查询 | 显示该学生住宿信息 |
| TC08 | 排序显示 | 点击按系/班级排序 | 表格按所在系、班级、学号显示 |
| TC09 | 提交调换申请 | 普通用户填写申请 | 生成待审核申请 |
| TC10 | 同意调换申请 | 管理员点击同意 | 申请状态变为已同意，学生住宿信息自动更新 |
| TC11 | 拒绝调换申请 | 管理员点击拒绝 | 申请状态变为已拒绝，学生住宿信息不变 |
| TC12 | 智能统计分析 | 管理员选择楼栋或宿舍 | 显示统计数据和运营建议 |

| TC15 | 待审床位保护 | 缩减容量时存在超范围待审床位 | 拒绝保存 |
| TC16 | 学生账号绑定 | 绑定不存在的学号 | 拒绝创建或更新账号 |
| TC17 | 会话失效 | 禁用、删除、改权或修改密码 | 相关旧会话立即失效 |
| TC18 | 报修流程 | 学生提交/撤回，管理员处理 | 状态与处理意见正确保存 |
| TC19 | 全列表分页 | 逐页打开住宿、申请、报修、床位、楼栋、宿舍、账号、日志和模型配置 | 显示总数、页码、翻页按钮和每页条数，数据不会集中在单页 |

## 5. 测试结论

系统核心功能、MySQL 持久化和 Web 接口均已验证，能够满足课程设计项目 5 的功能要求和企业化前端展示要求。
