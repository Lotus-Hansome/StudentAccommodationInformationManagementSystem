# 学生宿舍信息管理系统测试报告

## 1. 测试环境

- 操作系统：Windows
- JDK：23
- 数据库：MySQL 8.0
- 浏览器：Chrome/Edge 均可
- 默认访问地址：`http://localhost:8080`

## 2. 自动化测试

运行：

```powershell
.\run.ps1 -SmokeTest
```

预期输出包含：

```text
SMOKE_TEST_PASSED
```

本次已验证冒烟测试通过。

## 3. Web 接口验证

已验证：

- 管理员 `admin/admin123` 登录成功。
- `/api/overview` 可读取 MySQL 概览数据。
- `/api/students?mode=all` 可读取 MySQL 学生住宿列表。

验证结果：

```text
LoginSuccess=True
Role=ADMIN
TotalStudents=6
RoomCount=4
StudentRows=6
```

## 4. 功能测试用例

| 编号 | 测试内容 | 操作 | 预期结果 |
| --- | --- | --- | --- |
| TC01 | 管理员登录 | 输入 `admin/admin123` | 进入企业后台页面 |
| TC02 | 普通用户登录 | 输入 `student/student123` | 进入普通用户可见页面 |
| TC03 | 添加住宿信息 | 管理员填写完整学生住宿资料 | MySQL 保存成功，表格刷新 |
| TC04 | 修改住宿信息 | 管理员点击编辑并保存 | 对应学生记录更新 |
| TC05 | 删除住宿信息 | 管理员点击删除 | 记录从 MySQL 删除 |
| TC06 | 按宿舍查询 | 输入宿舍号查询 | 显示该宿舍学生 |
| TC07 | 按学号查询 | 输入学号查询 | 显示该学生住宿信息 |
| TC08 | 排序显示 | 点击按系/班级排序 | 表格按所在系、班级、学号显示 |
| TC09 | 提交调换申请 | 普通用户填写申请 | 生成待审核申请 |
| TC10 | 同意调换申请 | 管理员点击同意 | 申请状态变为已同意，学生住宿信息自动更新 |
| TC11 | 拒绝调换申请 | 管理员点击拒绝 | 申请状态变为已拒绝，学生住宿信息不变 |
| TC12 | 智能统计分析 | 管理员选择楼栋或宿舍 | 显示统计数据和运营建议 |

## 5. 测试结论

系统核心功能、MySQL 持久化和 Web 接口均已验证，能够满足课程设计项目 5 的功能要求和企业化前端展示要求。
